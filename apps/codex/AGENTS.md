# 個人設定

## 基本姿勢

### トレードオフスライダー

スコープ: 大
予算: 小
納期: 最小
品質: 最大

### 改善の約束は行動で示す

「意識します」「気をつけます」「メモリに残します」のような宣言は禁止。これらは次のセッションや別の環境に引き継がれず、実効性がない。

改善が必要な場合は、必ず以下のいずれかの行動を取ること:

- **AGENTS.md / CLAUDE.md にルールを追記する**（再発防止策をプロンプトに残す）
- **GitHub Issue を作成する**（修正が必要な箇所を記録する）
- **その場で修正する**（コード・設定・ドキュメントを直接変更する）

行動を伴わない反省文は出力しない。

## 対話ルール

### 判断要求の前置き

重要な判断（方針変更、リスクある操作、スコープ修正等）をユーザーに問う前は、必ず直前に「## まとめ」または「## 現状」セクションを出力する。
含めるべき情報:
- 背景（なぜ判断が必要になったか）
- 各選択肢の trade-off（リスク・コスト・期待結果）
- 推奨案がある場合はその根拠

## コマンド実行ルール

### commit メッセージ内のバッククォート禁止

commit メッセージ内のバッククォート（`` ` ``）は zsh OOM クラッシュを誘発するため絶対禁止。

- `-m "...` `` ` `` `code` `` ` `` `..."` のような直書きは bash がコマンド置換として展開し、zsh `command_not_found_handler` 無限再帰で fatal error: out of memory に至る
- 三連 backtick (` ``` ` fenced code block) も同様に危険
- 代替: コード片は引用符・インデント・タグ表記で代替するか、commit メッセージから除外する

### 外部スクリプト実行時のパイプの扱い

ファイル書き出しやログ生成を伴う外部スクリプトを実行する際、その出力に**バッファリング/切り詰めパイプを直接挟まない**。理由:

- 出力が見えなくなり、事後にエラー詳細やログを参照できなくなる
- `head` 等は SIGPIPE で上流プロセスを早期終了させる場合がある

**禁止例**:
- `script | head -N`
- `script | tail -N`
- `script > /dev/null`

**許容パターン**: `tee` で全出力をファイルに残してから表示を絞る。

- `script | tee /tmp/out.log | tail -n 50` — 全出力を `/tmp/out.log` に保存しつつ表示だけ tail に絞る
- 後で `cat /tmp/out.log` や `grep` で全文参照可能
- 標準エラーも残したい場合は `script 2>&1 | tee /tmp/out.log | tail -n 50`

## git 運用

### カレントディレクトリで実行する

gitコマンドは常にカレントディレクトリで実行する。`-C` オプションや絶対パス指定は使わない。

### force push の AI 代行実行

squashやamendして履歴をきれいにした場合、`git push --force-with-lease` の実行が必要であればユーザーに確認せずに実行する。
ただし、`--force` を使う場合・main 系ブランチへの push・squash 由来でない force push）はユーザーに確認を行う

### 外部公開コンテンツでのローカルパス取扱い

GitHub Issue / PR の本文・コメント、コミットメッセージ、公開リポジトリの Markdown、pastebin / gist 等の**外部公開コンテンツにはホーム配下絶対パス・認証情報・内部 host 名を直書きしない**。ユーザー名や環境情報が露呈する。

**禁止例**:
- `/Users/<name>/...`、`/home/<name>/...`、`C:\Users\<name>\...` 等のホーム配下絶対パス
- API トークン / 認証情報
- 内部 host 名 / 社内 URL

**推奨置換**:
- ホーム配下 → `~/...`
- リポジトリ配下 → repo-relative path（backtick 囲み、例: `` `bin/foo.sh` ``）
- 環境固有 prefix → プレースホルダ（例: `<homebrew-prefix>/bin/codex`）

**例外**: ローカル shell ログ / `/tmp/...` 中間ファイル等、外部公開しない作業ログは生パスのまま可。

**起票前チェック**: `gh issue create --body-file <f>` / `gh pr create --body-file <f>` 実行前に `grep -nE '/Users/|/home/[^/]+/|C:\\Users\\' <f>` で目視確認。

## 設定管理

### 設定ファイルのスコープ

- Global: `~/.codex/AGENTS.md` (このファイル / 全プロジェクト共通)
- Project: `<repo>/AGENTS.md` (リポジトリ内に存在する場合は優先)
- 一時上書き: `~/.codex/AGENTS.override.md` (通常は作らない。一時的に全体へ強く効かせたい場合のみ。完了後削除)
- Project 指示と Global 指示が衝突する場合は **Project を優先** する
- スコープが不明な場合は書き込む前にユーザーに確認すること
