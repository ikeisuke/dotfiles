# 個人設定

## シェルエイリアス回避ルール

この環境では `rm` `cp` `mv` に `-i`（確認プロンプト）がエイリアス設定されている。

Bashツールでこれらのコマンドを使用する際は、エイリアスを回避するためバックスラッシュを付ける：

```bash
# 正しい使い方
\rm file.txt
\cp src dst
\mv old new

# 避けるべき使い方（確認プロンプトで止まる）
rm file.txt
cp src dst
mv old new
```

## シェルコマンド置換 `$(...)` の回避

`$(...)` を含むコマンドはパーミッションの allow ルールにマッチしても
安全ヒューリスティクスにより毎回確認ダイアログが出る。
コマンド生成時は `$(...)` を使わない形式を優先すること。

- git commit は `-m "message"` または `-m "1行目" -m "2行目"` を使う
- 変数代入が必要な場合は複数ステップに分割する
- パイプ (`|`) やリダイレクト (`>`) は問題ない
- `$(...)` やバッククォート が確認対象になる

## セッションタイトル自動設定

以下のタイミングで `tools:session-title` スキルを使い、ターミナルタブのタイトルを設定する。

- **タイミング**: セッション開始時、ブランチ切り替え時
- **フォーマット**: `リポジトリ名 / ブランチ名`
- 個別に指定がある場合はそれに従う
